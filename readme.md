## 0. 目的
アクセス交換を支援するアンテナサイトを運用し、
アクセス数を広告収益などのお金に変換する効率が最も高いジャンルを確立する。
同時に、登録サイトが少ない初期フェーズでも、サイト運営者が登録したくなる仕組みを設計する。

## 2. ジャンル戦略（現時点の結論）

結論（一次）:
最初に攻める主軸は IT・SaaS・AIツール。

理由:
更新需要が高く、アンテナ形式と相性が良い
比較・導入記事へ自然に送客しやすい
収益化導線（トライアル/課金）を設計しやすい

補助ポートフォリオ（拡張時）:
ガジェット・家電（セール連動）
金融・節約（ただし法務・表現統制が整ってから）

実務上の最終判断ルール:
特化運用で4ゲートの通過率を確認し、
通過できない場合のみ総合型への拡張を検討。

## 3. 登録インセンティブ設計（初期フェーズ）

### 3-1. 登録理由（少数登録時でも成立させる）
流入保証の代わりに露出保証（最低インプレ）
Founding 30の先行優遇（固定枠、優先ローテ、初期パートナー表示）
無料の可視化レポート（imp/click/CTR/時間帯別）
改善提案付きの7日伴走

### 3-2. 悪用対策
品質審査（自動+手動5分）
無効クリック除外（IP/UA/間隔/Ref整合）
品質スコアで露出重みを調整（即BANより減衰運用）
NGワード/釣り見出し率で自動保留

### 3-3. オンボーディング
Step1 申請フォーム（最小項目）
Step2 自動審査（即時）
Step3 手動ライト審査
Step4 初期設定ウィザード
Step5 初回掲載通知
Step6 7日レビュー

## 6. 現時点の確信度

確信度: 高（ただし100%ではなく、ゲート検証で確定）
理由:
戦略案 -> 初期獲得設計 -> リスクレビューまで往復済み
実行不能点をMVP縮小で吸収済み
成功判定を数値ゲート化済み

補足:
「100%の自信」は実運用データがない段階では論理上保証不可。
そのため本計画は、100%断言の代わりに、
不確実性を4つのGo/No-Goゲートで潰していく設計とした。
