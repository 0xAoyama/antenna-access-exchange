# 申請フォーム仕様・審査基準・通知テンプレ（運営用）

最終更新日: 2026-05-30
運用SLA: 申請受領から24時間以内に初回判定連絡（承認/差し戻し/一時停止）

## 1. 申請フォーム項目定義

| 項目 | 必須 | 形式 | バリデーション |
|---|---|---|---|
| 申請区分 | 必須 | 個人/法人 | enum一致 |
| 氏名（法人は担当者名） | 必須 | 文字列 | 1〜80文字 |
| 氏名カナ | 必須 | 全角カナ | カナのみ |
| 法人名（法人のみ） | 条件必須 | 文字列 | 1〜120文字 |
| メールアドレス | 必須 | email | RFC準拠、重複不可 |
| 電話番号 | 必須 | 数字 | 10〜11桁 |
| 郵便番号 | 必須 | 数字 | 7桁 |
| 住所 | 必須 | 文字列 | 5〜200文字 |
| 生年月日（個人） | 個人必須 | YYYY-MM-DD | 年齢要件 |
| 法人番号（法人） | 法人必須 | 数字 | 13桁 |
| 本人確認書類 | 必須 | jpg/png/pdf | 10MB以内 |
| 利用目的 | 必須 | 自由記述 | 20〜1000文字 |
| 取扱予定カテゴリ | 必須 | 複数選択 | 禁止カテゴリ拒否 |
| 月間想定件数 | 必須 | 数値 | 1〜1,000,000 |
| 利用規約同意 | 必須 | チェック | true |
| 反社排除同意 | 必須 | チェック | true |

実装ルール
フロント/サーバ双方でバリデーション
申請ID採番（APP-YYYYMMDD-XXXX）
ファイルウイルススキャン
監査ログ保存（審査者・理由・時刻）

## 2. 審査基準

判定区分
通過（承認）
差し戻し（修正で再申請可）
停止（一時停止/保留、追加精査）

判定マトリクス

| 観点 | 通過 | 差し戻し | 停止 |
|---|---|---|---|
| 本人/法人確認 | 書類一致・有効 | 不鮮明/軽微漏れ | 偽造疑い/名義不一致 |
| 申請整合性 | 整合 | 誤記レベル不一致 | 複数項目矛盾/虚偽疑い |
| 利用目的 | 規約適合 | 説明不足 | 禁止商材/違法疑い |
| 過去履歴 | 問題なし | 軽微違反歴あり | 重大違反歴/再発傾向 |
| リスク評価 | 低〜中 | 中（追加説明で低減可） | 高（法務確認要） |

## 3. ペナルティ段階

| 段階 | トリガー | 措置 | 解除条件 |
|---|---|---|---|
| Stage0 注意喚起 | 軽微違反1回 | 注意メール | 修正完了 |
| Stage1 警告 | 軽微違反反復 | 警告＋期限付き是正 | 期限内是正 |
| Stage2 制限 | 中程度違反 | 一部機能停止 | 追加資料提出・承認 |
| Stage3 一時停止 | 重大違反疑い | 停止・調査 | 調査完了・承認 |
| Stage4 強制解約 | 違法/悪質/再発 | 契約終了 | 原則なし |

## 4. 通知テンプレ

4-1 承認
件名: 【審査結果】申請承認のお知らせ（{{申請ID}}）
本文: 申請ID {{申請ID}} は承認となりました。利用規約順守のうえご利用ください。

4-2 差し戻し
件名: 【要対応】申請内容の修正依頼（{{申請ID}}）
本文: 以下理由により差し戻しです。{{差し戻し理由}}。{{期限日}} までに再提出してください。

4-3 一時停止
件名: 【重要】アカウント一時停止のお知らせ（{{申請ID}}）
本文: 確認事項発生のため一時停止しています。追加資料提出をお願いします。

4-4 再開
件名: 【ご連絡】利用再開のお知らせ（{{申請ID}}）
本文: 確認完了により {{再開日}} 付で利用再開しました。

## 5. 24時間以内オンボードSLA

T+0h 自動受付メール・審査キュー投入
T+2h 自動チェック完了
T+12h 一次審査完了
T+20h 二次審査完了（必要時）
T+24h 初回判定通知送信

エスカレーション
T+18h 未判定でリーダー通知
T+22h 未判定で責任者通知
SLA超過時は原因分類と是正記録を必須化

例外
追加資料待ちは保留扱いで時計停止可
ただし初回判定通知自体は24時間以内に送信（差し戻し/一時停止で可）
